# /mnt/kimi/ Analysis
## User Data Mount Point

---

## Executive Summary

`/mnt/kimi/` serves as the primary user data mount point in the Kimi agent system. It provides organized storage for user uploads, AI outputs, and internal state persistence.

---

## 1. Directory Structure

```
/mnt/kimi/
├── upload/                # User file uploads (read-only for AI)
├── output/                # AI file outputs (write-only)
└── .store/                # Internal state (citation.jsonl)
```

---

## 2. Directory Purposes

### 2.1 upload/

**Purpose**: User file uploads

**Access**: Read-only for AI

**Contents**: Files uploaded by users for processing

**Examples**:
- Documents for analysis
- Images for processing
- Data files for transformation

### 2.2 output/

**Purpose**: AI-generated outputs

**Access**: Write-only for AI

**Contents**: Files generated by the AI system

**Examples**:
- Generated documents
- Processed images
- Analysis reports

### 2.3 .store/

**Purpose**: Internal state persistence

**Access**: System internal

**Contents**:
- `citation.jsonl`: Citation tracking for web sources

---

## 3. Inter-Module Relationships

```
/mnt/kimi/
    ├── upload/ ──► User uploads
    │               └── Processed by skills
    ├── output/ ──► AI outputs
    │               └── User download
    └── .store/ ──► citation.jsonl
                    └── Web source tracking
```

---

## 4. Security Model

| Directory | User Access | AI Access | Purpose |
|-----------|-------------|-----------|---------|
| `upload/` | Write | Read | Input files |
| `output/` | Read | Write | Output files |
| `.store/` | None | Read/Write | Internal state |
